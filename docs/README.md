# Probation Management System — Design Documentation

This folder contains design-phase documentation intended for project presentation and future engineering work.

## Documents (recommended order)
1. `docs/executive-summary.md` — one-page overview for stakeholders
2. `docs/system-overview.md` — goals, scope, assumptions, NFRs
3. `docs/architecture.md` — app/module architecture and request flows
4. `docs/data-model.md` — core entities + relationships (Mermaid ERD)
5. `docs/user-roles-permissions.md` — roles, permissions, and personalized views
6. `docs/workflows.md` — end-to-end workflows (Mermaid sequence diagrams)
7. `docs/security-privacy.md` — security model, privacy, auditability
8. `docs/deployment-operations.md` — env vars, runbooks, deployment notes
9. `docs/demo-script.md` — step-by-step demo path
10. `docs/slide-outline.md` — presentation slide outline

## Conventions
- Diagrams are written in Mermaid for easy rendering in GitHub/IDEs.
- The “source of truth” for model fields remains the Django models; docs describe intent and usage.
